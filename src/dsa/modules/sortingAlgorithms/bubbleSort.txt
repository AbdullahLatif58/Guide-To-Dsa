Bubble Sort – Complete Explanation

What is Bubble Sort?

Bubble Sort is a simple comparison-based sorting algorithm.

It repeatedly compares two adjacent elements and swaps them if they are in the wrong order.

After each pass, the largest unsorted element “bubbles” to its correct position at the end of the array.

⸻

Example

Input:

[5, 3, 8, 4, 2]

Output:

[2, 3, 4, 5, 8]

⸻

Core Idea

Imagine bubbles in water.

The biggest bubble always rises to the top.

Similarly, in Bubble Sort, the largest element moves to the end after every pass.

⸻

Step-by-Step

Array:

[5, 3, 8, 4, 2]

Pass 1

Compare:

5 and 3

Since

5 > 3

Swap

[3, 5, 8, 4, 2]

Compare

5 and 8

No swap.

Compare

8 and 4

Swap

[3, 5, 4, 8, 2]

Compare

8 and 2

Swap

[3, 5, 4, 2, 8]

Notice:

The largest element (8) has reached its correct position.

⸻

Pass 2

Ignore the last element because it is already sorted.

Current array:

[3, 5, 4, 2 | 8]

Compare

3 and 5

No swap.

Compare

5 and 4

Swap

[3, 4, 5, 2, 8]

Compare

5 and 2

Swap

[3, 4, 2, 5, 8]

Now the second largest element (5) is also in its correct position.

⸻

Pass 3

Current array:

[3, 4, 2 | 5 | 8]

Compare

3 and 4

No swap.

Compare

4 and 2

Swap

[3, 2, 4, 5, 8]

⸻

Pass 4

Current array:

[3, 2 | 4 | 5 | 8]

Compare

3 and 2

Swap

[2, 3, 4, 5, 8]

The array is now sorted.

⸻

Why do we use two loops?

Outer Loop

The outer loop counts the number of passes.

Each pass places one element in its final correct position.

For an array of size n, we need at most:

n - 1

passes.

⸻

Inner Loop

The inner loop compares adjacent elements.

Example:

arr[j]

and

arr[j + 1]

If

arr[j] > arr[j + 1]

swap them.

⸻

Why does the inner loop become shorter every pass?

Suppose:

[5,3,8,4,2]

After Pass 1:

[3,5,4,2,8]

The last element (8) is already sorted.

There is no need to compare it again.

After Pass 2:

[3,4,2,5,8]

Now both 5 and 8 are fixed.

Each pass reduces the unsorted portion of the array by one element.

⸻

Swap Condition

Only swap when

arr[j] > arr[j + 1]

Otherwise leave the elements unchanged.

⸻

Time Complexity

Worst Case:

O(n²)

Average Case:

O(n²)

Best Case (optimized with a swap flag):

O(n)

⸻

Space Complexity

O(1)

Bubble Sort is an in-place sorting algorithm because it does not require another array.

⸻

Stable Sorting

Bubble Sort is a stable sorting algorithm.

Equal elements keep their original order after sorting.

⸻

When is Bubble Sort used?

Bubble Sort is rarely used in production because faster algorithms exist.

However, it is excellent for learning:

* Nested loops
* Swapping elements
* Array traversal
* Sorting fundamentals
* Time complexity analysis

Understanding Bubble Sort makes algorithms like Selection Sort, Insertion Sort, Merge Sort, and Quick Sort much easier to learn.

⸻

Bubble Sort Algorithm

1. Start from the first element.
2. Compare the current element with the next element.
3. If the first is greater than the second, swap them.
4. Continue until the end of the unsorted portion.
5. The largest element is now in its correct position.
6. Repeat the process for the remaining unsorted elements.
7. Stop when the entire array becomes sorted.

⸻

Key Concept to Remember

Think of Bubble Sort like this:

Every pass pushes the largest remaining element to the end of the array.

That’s why it’s called Bubble Sort—the largest element “bubbles” to the top (the end of the array) after each pass.
